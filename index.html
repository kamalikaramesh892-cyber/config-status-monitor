🔄 Change Control Process

Change Management
1
Change Request
Developer identifies need for change and creates a new branch or commit with clear message describing the change
2
Code Review
Collaborator reviews the change on GitHub, checks for errors and ensures it meets project requirements
3
Approval & Merge
Change is approved and merged into main branch. Git tracks the exact time, author and files changed
4
Audit Update
Python script is run to regenerate audit log. Website is updated to reflect latest configuration status
